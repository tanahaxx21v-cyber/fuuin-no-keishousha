# 封印の継承者 — パワポケ4スタイル ルールブック

パワプロクンポケット4（GBA/2002年）裏サクセス「RPG風ファンタジー編」のUIを徹底分析し、本ゲームが守るべき設計ルールを定義する。

---

## 1. パワポケ4 RPGモードの核心的特徴

### 1.1 プラットフォームと画面サイズ
- GBA (240×160px) → ブラウザ版は縦長スマホ画面を想定
- ピクセルアート（ドット絵）スタイル：2〜3頭身のちびキャラクター
- 限られた解像度で表情豊かなキャラクター表現

### 1.2 マップシステム（すごろく形式）
- フィールドはフリーロームではなく **すごろく形式**（マス目を移動）
- 各ロケーション（マス）はアイコンで表示
- 現在地・行けるマス・行けないマスが色分けされている
- 移動はボタン選択で「N日かけて移動」という形式
- マップには地形に応じた色彩（草原・砂漠・山岳・海岸）
- 実装方針: **大きいドットアイコン + 接続線** のすごろくマップ

### 1.3 通常画面（フィールドシーン）
- 背景: ロケーションに合わせたイラスト背景（草原・町・洞窟など）
- キャラクタースプライト: 画面中央〜左に2〜3頭身のちびキャラが立つ
- メッセージボックス: **画面下部に濃紺色の枠付きボックス**
  - 背景色: `#080f38`
  - 枠線色: `#2848c0`（明るいインディゴブルー）
  - テキスト: 白・太字
  - 続きを示す ▼ カーソルあり
- ステータスバー: 画面上部に残り日数・HP・ゴールドを表示

### 1.4 バトル画面
- **背景**: 空（上部60%）+ 草地（下部40%）の2色構成
  - 空: グラデーション（明るい水色 `#5ab4e8` → 薄い水色 `#90d0f0`）
  - 草: 明るい緑 `#70c840` → 深い緑 `#287010`
  - 雲は白い楕円で blur表現
- **左側（プレイヤー/仲間エリア）**: 画面左20〜30%
  - 主人公スプライトが最も大きく（80px程度）
  - 仲間スプライトはやや小さく並ぶ（52px程度）
  - 各キャラの下にHPバーを表示
- **右側（敵エリア）**: 画面右60%
  - 敵スプライトを大きく表示
  - ボス戦では更に大きく、迫力のある演出
- **ステータスバー（上部）**: HP/MPバー + ターン数 + 行動者名
- **メッセージボックス（下部）**: 通常画面と同じ濃紺スタイル
  - バトルログを最新2〜3行表示
  - 行動中はアクター名 + 「のターン...」表示
- **コマンドグリッド**: 4つのコマンド（こうげき/スキル/どうぐ/にげる）
  - ボタンは正方形に近い形、アイコン + テキスト
  - 各コマンドは色分け（攻撃=赤系、スキル=紫系、etc）

### 1.5 キャラクタースプライト表示ルール
- **characters.jpg仕様**（1988×1194px、7列×2行）
  - タイトルバー: 上部約70px
  - 行0（レオン〜フィン）: y=70px〜y=632px（高さ562px）
  - 行1（ヴァイス〜ゼノ）: y=632px〜y=1194px（高さ562px）
  - 各列幅: 1988/7 ≈ 284px
  - 各セル内部: 上部約60%がキャラクターアート、下部40%が説明テキスト
- **表示方法**: `<img>` + `overflow:hidden` ラッパー方式
  - imgWidth = cols * size（7 * size でスケール）
  - imgX = -col * size
  - imgY = -(titleBar * scale + row * rowHeight * scale)
  - titleBarPx = 70（元画像px）
  - rowHeightPx = 562（元画像px）

### 1.6 色彩・デザイン指針
| 要素 | 色 |
|------|-----|
| 画面背景 | `#07071a`（深い紺色） |
| UIパネル | `#0c0c24`（やや明るい紺） |
| バトルUI背景 | `#12123a` |
| メッセージボックス | `#080f38` |
| 枠線（メイン） | `#2848c0`（インディゴ） |
| 枠線（ゴールド） | `#c07020` |
| 強調テキスト | `#ffd700`（ゴールド） |
| HPバー（高） | `#40c030` → `#208018` |
| HPバー（中） | `#c8b020` |
| HPバー（低） | `#e03020` → `#901010` |
| MPバー | `#3060e0` → `#1030a0` |

### 1.7 フォントとテキスト
- 全体: `font-family: monospace` または等幅フォント
- 見出し: `font-black`（最大太さ）
- ゲームテキスト: `font-bold`（太字）
- サイズ: モバイルファーストで読みやすいサイズ（最小10px）
- テキストシャドウ: 背景が明るい場合 `0 1px 3px #000`

## 2. BGM/音楽ルール

### 2.1 サウンドスタイル
- **チップチューン（8ビット）スタイル** — GBAゲームに合わせた電子音
- Web Audio APIで生成（外部ファイル不要）
- オシレーター型: `square`（矩形波）がメイン
- 音量は控えめに（masterGain: 0.25）

### 2.2 BGM種類
| シーン | BGM | テンポ | キー |
|--------|-----|-------|-----|
| フィールド・マップ | 明るく冒険的 | BPM130 | Gメジャー |
| 通常バトル | 緊張感・疾走感 | BPM168 | Eマイナー |
| ボスバトル | 重厚・ドラマチック | BPM185 | Cマイナー |
| タイトル | 壮大 | BPM120 | — |

### 2.3 効果音（SFX）種類
- `sfxAttack()` — 攻撃時（鋸波、短く鋭い）
- `sfxSkill()` — スキル使用（上昇アルペジオ）
- `sfxHeal()` — 回復（三角波、柔らかく上昇）
- `sfxDamage()` — ダメージ（鋸波、低く）
- `sfxVictory()` — 勝利ファンファーレ（短い旋律）
- `sfxDefeat()` — 敗北（下降する悲しい音）
- `sfxLevelUp()` — レベルアップ（上昇音）
- `sfxMenuSelect()` — メニュー選択（短いビープ）

## 3. 禁止事項（やってはいけないこと）

1. **現実的な3D・高解像度グラフィック** — ピクセルアート・チビキャラスタイルを維持
2. **テキストのないドットのみのマップ** — ホバーで名前表示は必要（ただし常時表示は不要）
3. **バトル中のHP初期値が減っている状態** — バトル開始時は全員MAXHPでUI表示してから処理
4. **パーティ管理画面** — 仲間は自動加入・死ぬまでパーティにいる
5. **キャラクターなしの戦闘画面** — 主人公・仲間のスプライトは必ずバトル左に表示
6. **音楽なしのゲーム** — 全フェーズにBGM、アクションにSFX必須
7. **メッセージボックスが画面上部にある** — 常に画面下部
8. **購入・課金操作** — 絶対禁止

## 4. 実装チェックリスト

毎回のデプロイ前に確認：
- [ ] バトル開始時に全員MAXHPで表示されるか
- [ ] キャラクタースプライトがバトル左側に表示されているか
- [ ] 敵スプライトがバトル右側に表示されているか
- [ ] メッセージボックスが画面下部の濃紺スタイルか
- [ ] BGMが各フェーズで再生されるか
- [ ] SFXが各アクションで鳴るか
- [ ] マップで現在地・移動可能地点が分かるか
- [ ] モバイルで表示が崩れていないか
